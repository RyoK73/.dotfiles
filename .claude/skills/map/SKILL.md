# Map モード

## 概要

情報を整理し、本当の課題・目標を再定義して、達成可能な形で保存する。

`/think` が「なぜ・どうすれば」で解決策を深掘りするのに対し、
`/map` は情報を整理し直し「本当の課題・目標は何か」を見つけることが目的。

## 入力の確認

`$ARGUMENTS` を確認する。

- **`@ファイル`が指定されている場合**: Read ツールでファイルを読み込み、内容を把握してからフェーズ1を開始する。読む前に「〜を確認する」と一言添える。
- **引数なしの場合**: 「今、頭の中にあるモヤや気になっていることを教えて」から始める。

---

## フェーズ1: 整理

### 目的

与えられた情報（ファイルまたは会話）を整理し、不足している情報をなくす。

### 進め方

1. 入力内容を自分なりに要約して示す
2. 不足・曖昧な点を特定し、一つずつ質問して補完する

質問の例:
- 「それはいつ頃から気になってる?」
- 「今それを動けない理由は何?」
- 「他に関連して気になってることはある?」
- 「それが解決したらどんな状態になってる?」

情報が揃ったと判断したら、整理した内容をユーザーに確認してフェーズ2へ進む。

---

## フェーズ2: 構造化・課題の再定義

### 目的

整理された情報をもとに、本当の課題・目標を見つけ出す。

### 進め方

以下の軸で構造化し、「本当にやりたいこと・解決すべきことは何か」を問い直す:

- **背景**: どういう文脈・経緯でこの課題が生まれたか
- **本当の課題・目標**: 表面的な課題の奥にある、本質的な目標
- **なぜやるか**: 目的・動機の結論
- **何をするか**: 具体的なアクション
- **完了条件**: Done の定義

課題が複数ある場合は、分割・統合の判断を都度行う。
機械的に分割せず、文脈から判断して「これは一個にまとめる? 分ける?」と聞く。

構造化した内容をユーザーに確認してフェーズ3へ進む。

---

## フェーズ3: 保存

「これを保存していい?」と確認してから保存する。

### 3-1: 会話ドキュメントの保存

会話の内容をマークダウンとして `~/dev/LIFE/note/` に保存する。

ファイル名: `YYYY-MM-DD-{テーマのスラッグ}.md`

フォーマット:

```markdown
---
date: {YYYY-MM-DD}
tags: [map]
---

# {テーマ}

## 整理した情報

{フェーズ1の内容}

## 課題・目標の再定義

{フェーズ2の構造化内容}
```

### 3-2: Issue作成

#### 事前チェック

```bash
gh auth status
```
失敗した場合: "gh CLI がインストールされていないか、認証されていません。`gh auth login` を実行してください。"

```bash
git remote get-url origin
```
失敗または `github.com` を含まない場合: "origin が GitHub リポジトリを指していません。"

#### 既存Issueの確認

```bash
gh issue list --state open --limit 100 --json number,title,labels,body
```

重複・関連するIssueがあれば確認してから作成する。

#### Issueのbodyフォーマット

```
### 背景

{どういう文脈・経緯でこの課題が生まれたか}

### 本当の課題・目標

{フェーズ2で再定義した本質的な目標}

### なぜやるか

{目的・動機の結論}

### 何をするか

{具体的なアクション}

### 完了条件

{Done の定義}
```

#### 作成コマンド

```bash
gh issue create \
  --title "{タイトル}" \
  --body "{上記フォーマットで生成した body}" \
  --label "life"
```

ラベル `life` が存在しない場合は作成する:
```bash
gh label create "life" --color "#0075ca" --description "人生・キャリアタスク"
```

#### 完了報告

```
## 保存完了

- ドキュメント: ~/dev/LIFE/note/{ファイル名}
- Issue: #{番号} {タイトル} {URL}
```

---

## 禁止事項

- 一度に複数の質問を並べない
- ユーザーが確認していないのにドキュメント・Issueを作成しない
- 合意なく会話を締めくくらない
- 「では実装しましょう」と言わない（このコマンドはタスク管理専用）
