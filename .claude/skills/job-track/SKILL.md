# 応募・選考ステータス管理

転職活動の応募・選考状況を記録・更新する。

## 引数の解析

`$ARGUMENTS` の形式:
- 空 → ステータス一覧を表示
- `{企業名}` のみ → その企業の詳細を表示
- `{企業名} {アクション}` → ステータスを更新

アクション一覧:
| アクション | 意味 |
|-----------|------|
| `watch` | 気になる（ウォッチリストに追加）|
| `apply` | 書類応募した |
| `screening` | 書類選考中 |
| `passed` | 書類選考通過 |
| `interview1` | 一次面接 |
| `interview2` | 二次面接 |
| `final` | 最終面接 |
| `offer` | 内定 |
| `reject` | 不合格 |
| `withdraw` | 辞退 |

---

## ケース A: 引数なし → 一覧表示

`~/dev/LIFE/career/applications.md` を読み込み、以下の形式で出力する：

```
# 選考ステータス一覧（YYYY-MM-DD）

## アクティブ（N社）
| 企業名 | ステージ | 最終更新 | 次のアクション |
|--------|---------|---------|--------------|
| ...    | ...     | ...     | ...          |

## 完了（N社）
| 企業名 | 結果 | 日付 |
|--------|------|------|
| ...    | ...  | ...  |
```

---

## ケース B: 企業名のみ → 詳細表示

`~/dev/LIFE/career/companies/{スラッグ}.md` を読み込み、
その企業の選考状況セクションを詳しく表示する。

---

## ケース C: 企業名 + アクション → ステータス更新

### C-1. applications.md の更新

`~/dev/LIFE/career/applications.md` を読み込む。

**アクティブテーブルの更新ルール:**
- `watch` / `apply` / `screening` / `passed` / `interview1` / `interview2` / `final`:
  - アクティブテーブルに行が存在しなければ追加、あれば更新
  - ステージ名: アクション → 日本語ステージ名のマッピングを使う
    - `watch` → 気になる
    - `apply` → 書類応募済み
    - `screening` → 書類選考中
    - `passed` → 書類選考通過
    - `interview1` → 一次面接
    - `interview2` → 二次面接
    - `final` → 最終面接
  - 最終更新: $CURRENT_DATE
  - 次のアクション: 状況に応じて適切なアクションを提案

- `offer` / `reject` / `withdraw`:
  - アクティブテーブルから行を削除
  - 完了テーブルに追加（結果: 内定 / 不合格 / 辞退）

### C-2. 企業プロファイルの更新

`~/dev/LIFE/career/companies/{スラッグ}.md` が存在する場合、
選考状況テーブルに日付・ステップ・メモを追記する。

存在しない場合は「/job-research {企業名} でプロファイルを先に作成することをお勧めいたします」と案内する。

### C-3. GTD への連携提案

面接系アクション（`interview1`, `interview2`, `final`）の場合:
「面接日程はお決まりですか？ /capture で GTD inbox に登録すると管理しやすくなります」と提案する。

### C-4. 報告

```
✅ ステータスを更新いたした

企業名:   {企業名}
ステージ: {以前のステージ} → {新しいステージ}
日付:     {$CURRENT_DATE}

{アクションに応じた一言コメント（武士言葉で）}
```
