# Curriculum モード

## 概要

学習ロードマップの表示・更新・進捗記録を行う。
`~/dev/LIFE/career/curriculum/` 配下のカリキュラムファイルを管理する。

## 手順

### Step 1: 引数の確認

`$ARGUMENTS` を確認する。

- **引数なし**: 現在のカリキュラムを表示する（Step 2へ）
- **引数あり**: 内容に応じてStep 2→更新へ進む

### Step 2: カリキュラムファイルの確認

```bash
ls ~/dev/LIFE/career/curriculum/
```

ファイルが存在する場合は読み込んで現在の状態を表示する。
存在しない場合は「カリキュラムファイルが見当たりません。新しく作成しましょうか？」と問う。

### Step 3: 現状の表示

カリキュラムを以下の形式で表示する:

```
## 現在のカリキュラム

### 目標
{最終目標}

### フェーズ
| フェーズ | 内容 | 状態 |
|---|---|---|
| Phase 1 | ... | 完了 / 進行中 / 未着手 |

### 今週のフォーカス
{現在取り組んでいること}

### 積み残し
{後回しにしている項目}
```

### Step 4: 更新（引数がある場合）

`$ARGUMENTS` の内容に応じて：

- **進捗更新**: 完了・進行中・未着手のステータスを変更
- **フェーズ追加**: 新しい学習フェーズを追加
- **目標変更**: ロードマップ全体の方向性を更新

更新内容をユーザーに確認してからファイルを更新する。

### Step 5: 保存

`~/dev/LIFE/career/curriculum/roadmap.md` に保存する（ファイルが存在しない場合は新規作成）。

更新後は「カリキュラムを更新いたした」と報告する。

## カリキュラムファイルのフォーマット

```markdown
---
updated: YYYY-MM-DD
goal: {最終目標}
---

# 学習カリキュラム

## 最終目標

{何を目指しているか}

## フェーズ

### Phase 1: {フェーズ名}
- 期間: {期間}
- 内容: {学ぶこと}
- 状態: {完了 / 進行中 / 未着手}

## 今週のフォーカス

{現在取り組んでいること}

## 参考リソース

- {書籍・URL・コース名}

## 積み残し

- {後回しにしている項目}
```
